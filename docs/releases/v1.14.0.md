## Summary

Consumes the himitsubako v0.7.0 (HMB-S031) SOPS fix and exposes its new
`age_identity` and `sops_config_file` kwargs through OWB's `SecretsConfig`,
factory, and wizard. Users with non-default sops/age setups can now configure
decryption entirely in `workspace.yaml` without having to export
`SOPS_AGE_KEY_FILE` in every shell.

## Upgrade Notes

- Existing `workspace.yaml` files continue to work with zero changes. New fields
  default to `None` and preserve the env-var fallback for `SOPS_AGE_KEY_FILE`.
- If you run SOPS with a non-default age key location, you can now declare it:

  ```yaml
  secrets:
    backend: sops
    sops_secrets_file: .secrets.enc.yaml
    sops_age_identity: ~/keys/age/owb.txt
    sops_config_file: ~/keys/.sops.yaml
  ```

- Configured paths that point at non-existent files fail loudly at backend
  construction with `FileNotFoundError`. This is intentional â€” silent fallback
  to the default would mask misconfiguration.
