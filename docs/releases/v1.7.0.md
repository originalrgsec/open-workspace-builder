# v1.7.0 — SBOM Enrichment: Provenance, Capability, License (OWB-S107b)

**Released:** 2026-04-11
**Sprint:** 21
**Tests:** 1,723 (+77 new S107b tests, from 1,646)
**Theme:** Enrich the S107a SBOM substrate with provenance, declared
capabilities, and license cross-reference for SSCA-grade inventories.

## Code

| Module | Change |
|--------|--------|
| `src/open_workspace_builder/sbom/license.py` | **New** — license detection module: `LicenseEntry`, `LicenseSource`, `AllowedLicensePolicy`, `detect_license`, `identify_spdx`, `load_allowed_licenses`. SPDX identification by distinctive-phrase fingerprinting (case- and whitespace-normalized) over 13 well-known licenses. |
| `src/open_workspace_builder/sbom/capability.py` | **New** — capability extraction for skills/agents/commands and MCP servers. `Capability`, `CapabilityKind`, `parse_allowed_tools`, `extract_skill_capabilities`, `extract_mcp_server_capabilities`. **MCP env values are never recorded** — keys only, enforced by a dedicated test. |
| `src/open_workspace_builder/sbom/provenance.py` | **New** — provenance detection: `Provenance`, `ProvenanceType`, `ProvenanceConfidence`, `InstallRecord`, `detect_provenance`, `read_install_record`. Four-source priority order (frontmatter → install record → git history → local) with confidence scoring. Git remote URLs normalized to canonical https form. Install record reader is forward-compatible: locked schema, no writer yet. |
| `src/open_workspace_builder/sbom/discover.py` | `Component` extended with three optional fields (`provenance`, `capabilities`, `license`) defaulting to empty so S107a callers/output remain unchanged. Discovery functions now thread the loaded `AllowedLicensePolicy` through and call enrichment per component. |
| `src/open_workspace_builder/sbom/builder.py` | Emits new `owb:provenance:*`, `owb:capability:*`, and `owb:license:warning` properties; spec-native `licenses` field on every component; top-level `owb:license:non-allowed-count` aggregate. New `count_non_allowed_licenses(bom)` helper for the CLI exit code. |
| `src/open_workspace_builder/cli.py` | `owb sbom generate` now exits with code 2 when one or more components have a non-allowed or unrecognized license, prints the count to stderr, and documents the new exit-code contract in `--help`. |
| `src/open_workspace_builder/data/allowed_licenses.toml` | **New** — runtime authority for the license cross-reference, machine-readable twin of vault `Obsidian/code/allowed-licenses.md`. Allowed/conditional/disallowed buckets with SPDX IDs. |

## Notes

- **Session budget check (S117 / 8-pt cap):** S107b delivered in ~5 pt of
  story work plus ~1-2 pt of meta-work (sprint planning, backlog scrub for
  S118/S119, SEC-003 quarantine deferral). Total load ~6-7 pt, comfortably
  under the 8-point cap. No context pressure symptoms observed.
- **Integration verification per integration-verification-policy §1:** the
  new `tests/sbom/test_workflow_ac_s107b.py` exercises the full
  `owb sbom generate` CLI pipeline end-to-end through Click's `CliRunner`
  on a four-case fixture workspace covering each enrichment branch
  (frontmatter explicit source, git history with origin remote, local
  fallback in a non-committed file, GPL-3.0 sibling LICENSE). All four
  cases assert the expected provenance type, capability properties, and
  license cross-reference outcome. Exit code 2 contract for non-allowed
  licenses is verified at the CLI boundary.
- **Hash stability discipline (the central S107b promise):** the new
  `test_s107a_hash_stability_preserved` regression test in
  `tests/sbom/test_example_fixture.py` carries the v1.6.0 `bom-ref` and
  content-hash values as a hard-coded baseline. If a future change to the
  `norm1` algorithm or `bom-ref` derivation logic ever breaks this test,
  the change must be shipped as `norm2` (or higher) rather than as a
  fix-the-test response. This is enforced by the test's failure message,
  not just by convention.
- **MCP env-value safety:** the SBOM is intentionally designed to be
  shared with external SSCA pipelines. Recording env *values* from
  `.mcp.json` would risk leaking API tokens, OAuth secrets, and credentials
  into shared artifacts. The capability extractor records only env *keys*,
  and `tests/sbom/test_capability.py::test_env_values_never_leak` asserts
  this invariant by serializing the entire capability output and checking
  that no known secret value substring appears in any property name or
  value. This test must remain green forever.
- The code lands as a single feature commit on `main` (matching the Sprint
  20 pattern). The version bump, CHANGELOG, release manifest, and SDR
  entry land in a separate `docs:` commit per the Sprint 18/19/20 pattern.
