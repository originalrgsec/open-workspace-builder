# v1.18.0 — Distribution Content Refresh (Sprint 34)

Released: 2026-04-22

## Summary

First bundled-content refresh since v1.14.1 (Sprint 30). The v1.15.0 → v1.17.0 arc shipped code, CI, and SBOM changes only; eight sprints of vault-policy, global-rule, and skill drift had accumulated outside the distribution. Sprint 34 ports the load-bearing pieces — refined autonomous SDLC, quality-gates policy, PII-audit skill with first-use interview, and two new PreToolUse hooks — into `src/open_workspace_builder/content/` and `src/open_workspace_builder/vendor/`. Every artifact carries a genericize pass so operator-specific paths, account names, project identifiers, and incident citations are removed before ship.

- **OWB-S147** (M): policy genericize batch 1 — targeted deltas onto the lean v1.14.1 baseline for `development-process.md` (scrub skills, quality-metrics table, "verify don't create" Release guidance, renumbered 10-item checklist), `oss-health-policy.md` (mandatory Pre-Install Gate with license-first sequencing), `allowed-licenses.md` (retroactive de-branding of an already-shipped policy), and a one-line addition to `product-development-workflow.md` for the scrub-skills pointer. Zero origin citations.
- **OWB-S148** (M): three new bundled policies — `scrub-skills-quick-reference.md`, `cli-standards.md`, `pii-handling-policy.md`. The PII policy restructures around a three-way storage-backend choice (himitsubako recommended / direct age / explicit refuse-plaintext) and a two-level profile location.
- **OWB-S149** (M): ECC rules genericize — ports the Autonomous End-to-End Contract, enumerated 10-item Sprint Completion Checklist, and Git Ops Ownership section into `vendor/ecc/rules/common/sprint-workflow.md`. `git-workflow.md` gains the Autonomous Git Operations + Gated Ops split. Small updates to `development-workflow.md` (sprint-execution shortcut) and `security.md`.
- **OWB-S150** (XS): agent drift sync — `code-reviewer`, `planner`, `tdd-guide` pick up policy-consultation sections referencing the bundled policy filenames.
- **OWB-S151** (M, absorbs TD-005): two new PreToolUse hooks bundled under `vendor/ecc/hooks/` — `security-writetime.py` (nine regex rules covering GHA injection, shell=True, unsafe yaml, verify=False, Flask debug, eval/exec, pickle.loads, 0.0.0.0 bind, hardcoded private keys) and `sprint-close-reminder.sh`. Warn-only semantics, `# noqa: security` suppression marker, new AD-19 ADR documenting the architecture choice and the declarative-loader follow-up.
- **OWB-S152** (M): new bundled `vault-pii-audit` skill with a first-run interview that configures the PII profile, chooses an encrypted-store backend, and refuses to emit a plaintext findings index. Scanner refactored to be config-driven: PII patterns gate on the profile's `categories:` list, exclusions merge on top of a minimal `noreply@anthropic.com`-only pre-seed.
- **OWB-S153** (S): release plumbing — CHANGELOG, this manifest, version bump, mkdocs nav.

## Deferred

- **TD-005 §2 declarative rule loader** for `security-writetime.py`. Rules stay as a `@dataclass(frozen=True)` tuple in this release; schema design (severity levels, rule kinds, user-extension surface) is a separate follow-up story. See AD-19 for the decision.
- **`owb-tag-story.sh` hook.** OWB-internal, stays unbundled.
- **JS / TS / Go rules** for `security-writetime.py`. Language scope remains Python + GitHub Actions + infrastructure config.
- **Blocking mode** for any `security-writetime.py` rule. Warn-only per AD-19.
- **Scanner unit tests** for `vault-pii-audit/scripts/scan.py`. Filed as a follow-up story candidate; the skill's SKILL.md prose is the primary contract.
- **`owb init` auto-scaffold of `.owb/pii-profile.yaml`.** The skill's first-run interview generates the file on demand; pre-populating an empty profile at init time would short-circuit the interview and degrade UX.

## Migration

No breaking changes. No Python API signatures changed. No CLI command signatures changed. The bundled content deploys to the same `code/`, `.ai/agents/`, `.ai/hooks/`, and `.skills/` paths as before; operators running `owb migrate` on an existing workspace will see the new and updated files land alongside the existing tree, with `.skills/vault-pii-audit/` as a net-new directory.

First invocation of the new `vault-pii-audit` skill triggers the one-time interview. Operators who already maintain an encrypted PII store using age / SOPS can answer "age" to the backend question and point the profile at their existing `.secure/pii_store.yaml.age` path.
