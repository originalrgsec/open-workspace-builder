# v1.8.0 — SBOM Operational Commands, SPDX 2.3, Quarantine, Docs (OWB-S107c)

**Released:** 2026-04-11
**Sprint:** 22
**Tests:** 1,814 (+91 new S107c tests, from 1,723)
**Theme:** Layer the operator-facing surface on top of the S107a substrate
and S107b enrichment so a workspace owner can answer the four operational
SBOM questions: what is in here, what changed, did the workspace drift,
and was anything added inside the supply-chain quarantine window. Plus
the user-facing documentation OWB has owed since v1.6.0.

## Code

| Module | Change |
|--------|--------|
| `src/open_workspace_builder/sbom/diff.py` | **New** — structural diff between two CycloneDX 1.6 SBOMs joined by `bom-ref`. `DiffResult`, `FieldChange`, `ChangedComponent` dataclasses; `diff_boms`, `load_bom`, `render_json`, `render_text`. Comparable surface intentionally narrow: content hash, license, capability set, provenance source/commit. `added_at` is **not** in the surface so cosmetic mtime drift never registers. |
| `src/open_workspace_builder/sbom/verify.py` | **New** — workspace drift verification. `VerifyOutcome`, `verify_workspace`, `default_canonical_path`. Regenerates the workspace SBOM in-memory and reuses `diff.py` against a canonical file (default `.owb/sbom.cdx.json`). |
| `src/open_workspace_builder/sbom/show.py` | **New** — read-only SBOM inspector. `ComponentSummary`, `ComponentDetail`, `summarize`, `find_component`, `render_summary_text`, `render_summary_json`, `render_detail_text`, `render_detail_json`. Fixed-column summary table by default; full property dump under `--component <bom-ref>`. |
| `src/open_workspace_builder/sbom/spdx.py` | **New** — hand-rolled SPDX 2.3 JSON emitter. **No new dependency.** `serialize_spdx`, `build_spdx_document`, `sanitize_spdx_id`, plus internal `_component_to_spdx_package`, `_capability_annotations`, `_spdx_license_fields`, `_download_location`, `_source_info`. Maps internal `Component` records to SPDX `packages` with `name`, `versionInfo`, `SPDXID` (sanitized via `[A-Za-z0-9.\-]+` regex with consecutive-hyphen collapse), `checksums`, `licenseConcluded` / `licenseDeclared`, `downloadLocation` from provenance, `sourceInfo` carrying `commit:<sha>` and `added-at:<date>`, capability annotations whose dates are threaded from the document `created` field for byte-stable output. |
| `src/open_workspace_builder/sbom/quarantine.py` | **New** — `added_at` window check. `QuarantinedComponent`, `QuarantineReport`, `check_quarantine`, `check_workspace_quarantine`, `render_report_text`, `render_report_json`. Walks an SBOM dict (or regenerates one in-memory from a workspace) and reports components whose `owb:provenance:added-at` falls inside a configurable window (default 7 days), mirroring the Python package quarantine policy from S089. Future dates are skipped (clock skew / fixture date safety). |
| `src/open_workspace_builder/sbom/provenance.py` | `Provenance` gains an `added_at` field. New `_detect_added_at` helper sources from `git log --diff-filter=A --follow --format=%aI` (high confidence) or file `mtime` (low confidence). `detect_provenance` threads `added_at` onto every returned record via `dataclasses.replace`. **Critically excluded from the normalized content hash by construction** — the hash function operates on file content, not metadata — so v1.6.0 and v1.7.0 component hashes remain byte-stable. The S107b hash-stability regression test enforces this against the v1.6.0 baseline. |
| `src/open_workspace_builder/sbom/builder.py` | `_provenance_properties` now emits the `owb:provenance:added-at` CycloneDX property when set. No other changes. |
| `src/open_workspace_builder/security/gate.py` | New `_check_skill_quarantine(workspace, days=7)` function returning a `GateCheck` for the scanner gate pipeline. Misconfigured workspaces and missing modules degrade to `passed=True` with a `skipped` detail rather than failing the gate. **Pre-existing latent fix:** `_check_quarantine` (the S089 package check, unrelated to S107c) had a bare `except ImportError` around an import + attribute lookup; widened to `(ImportError, AttributeError)`. Caught during code review of the new neighbor. |
| `src/open_workspace_builder/cli.py` | Four new `owb sbom` subcommands: `diff <old> <new>`, `verify [--workspace] [--against]`, `show <sbom> [--component]`, `quarantine [--workspace] [--days] [--sbom]`. All four use consistent `--workspace`/`--format` option naming per `cli-standards.md` and return the 0/1/2 exit codes from S107a. `owb sbom generate --format` now accepts `cyclonedx` or `spdx` (was CycloneDX-only with an "SPDX deferred to S107c" disclaimer). New scanner option `owb scan --skill-quarantine` wires `_check_skill_quarantine` into the scan gate pipeline behind a feature flag, **default off** so existing scan workflows do not break silently. |

## Notes

- **Session budget check (S117 / 8-pt cap):** S107c estimated at ~6 pt, delivered in a single session. Including sprint-close meta-work the total session load is ~7 pt, under the 8-pt cap. Code-reviewer subagent flagged one HIGH (pre-existing latent bug in `gate.py`, fixed in this sprint) and three MEDIUM items, two of which were addressed inline (SPDX byte-stability, SPDX ID hyphen collapse) and one deferred (`_property_value` helper consolidation across `diff.py` / `show.py` / `quarantine.py`, refactor work).
- **Integration verification per integration-verification-policy §1:** `tests/sbom/test_cli.py::TestWorkflowAcceptance::test_generate_show_diff_verify_quarantine_pipeline` runs the full `generate → show → verify → diff → quarantine → generate --format spdx` chain through Click's `CliRunner` on a fixture workspace, asserting every exit code along the way. This is the workflow-level AC for S107c. The pre-existing pipeline smoke test (9 tests) still passes.
- **Hash stability discipline (continued from S107b):** the v1.6.0 baseline in `tests/sbom/test_example_fixture.py::test_s107a_hash_stability_preserved` continues to pass under S107c regeneration. `added_at` is metadata and never enters the normalized content hash by construction. Any future change to the `norm1` algorithm or `bom-ref` derivation must still bump `norm1` → `norm2` rather than break this test.
- **MCP env-value safety (continued from S107b):** the new `diff.py`, `show.py`, and `quarantine.py` code paths only consume CycloneDX `properties` with names that already passed through the S107b extractor's key-only filter. Code review confirmed no new code path can surface env values from the SBOM.
- **Skill-quarantine scanner default:** the new `--skill-quarantine` flag is opt-in (`is_flag=True, default=False`). The gate hook is also defensive — missing module, missing workspace, and quarantine errors all degrade to `passed=True` with a `skipped` detail rather than failing the scan. Default-flip ships in a follow-on after a deprecation cycle.
- **SPDX byte-stability:** an early draft pulled `datetime.now()` inside the annotation builder, breaking byte-stability when components had capabilities. Fixed by threading the resolved `created` timestamp from `build_spdx_document` → `_component_to_spdx_package` → `_capability_annotations`. Regression test added at `tests/sbom/test_spdx.py::test_byte_stable_with_capabilities`.
- **Sprint cost summary (token consumption ledger):** the `2026-04-11` ledger entry rolls together both the Sprint 21 close-out (S107b release manifest, version bump, vault updates) and the full Sprint 22 execution (S107c implementation + close-out). Daily total $365.10 across 2,030,422 read tokens and 467,416,671 cache-read tokens. Sprint-22-only attribution is not separable from the daily roll because the two sprints landed back-to-back in the same session window.
- **Pre-existing latent fix shipped opportunistically:** during code review of `_check_skill_quarantine` the `python-reviewer` agent flagged a latent bug in the neighboring `_check_quarantine` (S089 package check) where a bare `except ImportError` around an import + attribute lookup would have silently degraded on a future refactor. Widened to `(ImportError, AttributeError)`. Out of S107c scope but cheap to fix while the file was open.
