# v1.16.0 — Security Follow-Through & Hook Install (Sprint 32)

Released: 2026-04-18

## Summary

Closes both HIGH security findings spawned by Sprint 31's
`/code-review` pass and installs the pyright pre-commit + CI gate
authorized by DRN-078.

- **OWB-SEC-005** (HIGH, CWE-918 / CWE-78 / CWE-20): validate
  `repo_url` before `git clone`. Prevents `file:///etc/passwd`
  exfiltration, `ssh://internal` SSRF, and `--upload-pack=evil`
  argument-injection.
- **OWB-SEC-006** (HIGH, CWE-1333): scanner input-length caps
  defuse ReDoS against the pattern registry. Full registry audit
  shipped: 0 exponential patterns, 9 quadratic, 49 linear.
- **OWB-S139**: pyright pre-commit + CI gate enforces the DRN-078
  frozen error budget of 96. Script-based gate makes budget changes
  traceable (DRN + diff required).

Sprint scope was deliberately split from the original 10-pt proposal.
Remaining Sprint 31 spawns (OWB-S141, S142, S143, S144) land in
Sprint 33 "Hook Hardening & SBOM Refactor".

## Migration notes

- **Default behavior unchanged** for existing production configs.
  Every OWB source config already uses `https://` URLs, which is
  the default scheme allowlist.
- If you have a source using `ssh://` or `git://`, add to your
  `config.yaml`:
  ```yaml
  sources:
    allowed_schemes:
      - https
      - ssh
  ```
- Scanner callers that assert exactly zero flags on oversized
  inputs (single-line > 16 KiB or whole-file > 1 MiB) will now see
  a `scan_limit` warning flag. Typical content is unaffected; caps
  are 10x normal size.
