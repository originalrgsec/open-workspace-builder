# v1.17.0 — Hook Hardening & SBOM Refactor (Sprint 33)

Released: 2026-04-18

## Summary

Closes every Sprint 31 spawn that did not make it into the Sprint 32
cut. Two security-MEDIUMs, one leaky abstraction / pyright win, and
one dependency pin bump now that himitsubako v0.9.0 landed on PyPI
the same day. Plus one pre-existing tz bug that surfaced during
Sprint 33 execution and was fixed inline.

- **OWB-S141** (XS): `himitsubako>=0.7.0` → `>=0.9.0` in all three
  `pyproject.toml` occurrences. S134 (Sprint 31) already verified
  zero public-API impact; this is the mechanical follow-through.
- **OWB-S142** (S, security): dependency gate distinguishes
  tool-crashed from tool-not-installed. Default `fail_closed=true`
  turns an errored wrapped tool into a hard fail instead of a
  silent pass — removes the "induce a tool error to clear the
  gate" attack path.
- **OWB-S143** (S, security): `_fetch_pypi_json` and `_query_osv`
  cap HTTP reads at 1 MiB. Defends against a compromised or
  spoofed endpoint returning a multi-megabyte payload.
- **OWB-S144** (M, refactor): `BomWithMetadata` dataclass replaces
  the earlier monkey-patching of `_owb_options` and
  `_owb_non_allowed_count` onto the vendored CycloneDX `Bom`.
  Pyright basic budget drops **96 → 40**.
- **Sprint 33 hotfix** (XS): `sbom/provenance` emitted UTC date
  for file mtime while `check_quarantine` compared against local
  `date.today()`. In evening US timezones this skew skipped
  freshly-created components. Fixed to use local tz on both sides.

## Security

### OWB-S142 — Dependency gate fail-closed on tool error

| File | Change |
|------|--------|
| `src/open_workspace_builder/security/gate.py` | New helper `_errored_check(name, exc, fail_closed)` shapes the errored path. `_check_pip_audit`, `_check_guarddog`, `_check_license`, `_check_quarantine`, `_check_skill_quarantine` all accept `fail_closed: bool = True` and route unexpected exceptions through the helper. `run_gate` threads the knob. |
| `src/open_workspace_builder/config.py` | `SecurityConfig.fail_closed: bool = True` (new field). |
| `tests/security/test_gate.py` | `TestGateFailClosedOnToolError` — 9 new tests covering errored vs not-installed vs fail_closed=False for each wrapped tool plus the SecurityConfig default and run_gate threading. |

**Behavior change:** If a wrapped tool raises an unexpected
exception (previously swallowed as a silent pass), the gate now
returns `passed=False` with `"errored: <exc-type>: <msg>"` details.
Tool-not-installed paths still return `passed=True, "skipped"`.
Operators who need the legacy skip-on-error semantics can set
`security.fail_closed: false`; the errored path then passes with
the detail labelled `"errored (fail_closed=false): ..."` and the
gate logs a WARNING line naming the knob.

### OWB-S143 — HTTP response size cap (1 MiB)

| File | Change |
|------|--------|
| `src/open_workspace_builder/security/quarantine.py` | New `MAX_PYPI_BYTES = 1_048_576` and scoped `FetchError`. `_fetch_pypi_json` accepts `max_bytes` kwarg; reads `max_bytes + 1` to detect truncation; catches `FetchError` alongside URLError/OSError to preserve "any error returns None" contract. |
| `src/open_workspace_builder/security/suppression_monitor.py` | Same pattern — `MAX_OSV_BYTES = 1_048_576`, scoped `FetchError`, `_query_osv` cap. `check_suppression` adds `FetchError` to its except tuple so callers see a clean status. |
| `tests/test_quarantine.py` | `test_pypi_fetch_truncates_oversize`, `test_pypi_fetch_accepts_small_payload`, `test_pypi_fetch_honours_custom_cap`. |
| `tests/security/test_suppression_monitor.py` | `TestOsvQueryHttpCap` — 3 tests covering oversize → `FetchError`, normal payload, custom cap. |

Real PyPI JSON bodies for a single package run 10–200 KB; real OSV
records run 2–30 KB. 1 MiB is ample headroom for the largest
legitimate packages (tensorflow, torch, many-versioned libraries).

## Hotfix

### SBOM mtime provenance — local tz, not UTC

Six SBOM tests (`test_quarantine`, `test_cli`) began failing
around 20:00 local on 2026-04-18 when the UTC date rolled forward
a day before local did. `sbom/provenance._detect_added_at` emitted
UTC date for file mtime while `sbom/quarantine.check_quarantine`
compared against `date.today()` in local tz. Freshly-created
components landed as "future-dated" and got skipped.

Fix: drop `tz=timezone.utc` from the mtime-fallback path. Local
tz matches `date.today()`. Added a regression test
(`test_added_at_uses_local_tz_not_utc`) that asserts
`prov.added_at == date.today().isoformat()` for a just-created
file.
