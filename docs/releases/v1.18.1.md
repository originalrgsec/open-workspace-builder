# v1.18.1 — Supply-Chain Policy Patch (Sprint 35)

Released: 2026-05-01

## Summary

Single-story patch release that closes a distribution-genericize gate slip on the bundled `supply-chain-protection.md` policy. A +18 line addition documenting an ADR-override procedure had landed (uncommitted on `main`) carrying cross-project framing — a CI Action concept ("`abop-shared-kb` `dependency-gate` GitHub Action"), an external story ID, and operator-account-specific repo framing — that did not match the artifact OWB actually ships. OWB's `dependency-gate` is a **local PreToolUse hook**, not a CI gate Action, so the section is reframed around the bundled `dependency-gate.py` hook installed at `~/.claude/hooks/dependency-gate.py` after `owb init`.

This is the same gate that Sprint 34 (S147–S150) enforced for the v1.18.0 content bundle — strip operator paths, account names, project-specific story IDs, federal/CISO framing — applied to a single follow-up section that slipped past the gate. No new functionality, no test changes.

Two prior commits past the v1.18.0 tag also ride along into v1.18.1: `c114f53` (binary-integrity in CI sections — `_checksums.txt` manifest verification or hardcoded SHA256 fallback; reusable-workflow SHA pinning to 40-char commit SHAs; `--exit-code 1` mandatory for direct `trivy fs` workflow calls) and `543ad69` (SDR sprint-plan backfill for Sprints 31–34). Both shipped clean policy content; only `c114f53`'s commit message carries an external origin citation, which is acceptable and immutable.
